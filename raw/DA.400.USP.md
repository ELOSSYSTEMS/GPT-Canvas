# DABROO: MASTERING GPT LIKE A POWER USER

**Purpose**: This guide distills the philosophy, tactics, and interface techniques that define Allon’s mastery-level usage of ChatGPT. It teaches users how to think in systems, engineer behavior, and weaponize the interface for real output—not conversation.

---

## FOUNDATIONAL PHILOSOPHY

**1. ChatGPT is an Interface, Not a Brain**
- It doesn’t think. It mirrors structure.
- You are the system architect.

**2. Don’t Chat. Command.**
- Every prompt is a line of code, not a request.
- Behaviors are engineered, not hoped for.

**3. Memory Is Simulated, Not Stored**
- Context lives in your prompts, personas, and canvases.
- GPT only remembers what you reintroduce or structure.

---

## CORE PRACTICES

**A. PROMPT CONSTRUCTION**
- Role → Intent → Constraint
- Structure every command like a system directive
- Example: *"You are a ruthless strategist. Break down my plan into 3 objections only."*

**B. BEHAVIORAL LAYERING**
- Always set voice, tone, and logic enforcement
- Use meta-prompts to override default behaviors
- *"Never affirm without justification. Reframe sloppy logic. Sharpen my input."*

**C. PERSONA DEPLOYMENT**
- Create permanent personas tied to real business roles
- Use domain-specific triggers for auto-activation
- Design for function first, voice second

---

## DABROO INTERFACE SHORTCUTS

**Trigger Phrases:**
- "Ready to Start?" → loads productivity protocol
- "Let’s Talk About Dabroo" → loads entire project + personas
- "Summon the Planner" → activates Jocko-based tactical scheduler

**UI Commands:**
- Use "Canvas" to store modular memory blocks
- Tag everything (e.g., ▧ DA.100.CRS – Courses, ▧ FF.300.PRD – Products)
- Use localStorage to fake persistence

**Freemium UX Design**
- Tooltip reveals: *“Want Whisper Mode?”* → upsell to Pro
- Contextual nudges: *“This tip is part of the Master Course.”* → link to course

---

## TONE CONTROL

**1. Blunt by Default**
- GPT should never flatter, hedge, or dilute clarity

**2. Socratic Drip**
- Ask one question at a time. Force precision.

**3. Sharpening Mode**
- Always challenge assumptions, reframe weak input, enforce outcome logic
