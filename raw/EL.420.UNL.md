## ELos Intermediate Tier Unlock Protocol

### 🧭 Unlock Criteria (Hybrid Formula)
User unlocks access to the Intermediate tier UI and gated tools upon completing **3 of the following 5 behavioral milestones**:

1. Opened 3 or more Phase 2 (Blue Leaf) Academy guides
2. Stored a memory instruction or created a canvas via prompt
3. Modified tone or behavior through a menu interface
4. Referred to or reused a past message or logic path
5. Viewed at least 2 Academy canvases and acted on one

All milestones are tracked silently and enforced structurally. No session count, timers, or artificial gating.

---

### 🧠 Experience Design – What Changes at Unlock
- No new menus appear; **previously visible grayed-out options become active**
- User receives no intrusive popup—only inline visual confirmation
- All Intermediate features become **clickable but gated with gentle upgrade prompts**

---

### 🪞 Feature Behavior Post-Unlock (Pre-Upgrade)
| Feature            | State Before Unlock       | State After Unlock (No Upgrade) | State After Upgrade       |
|--------------------|----------------------------|----------------------------------|----------------------------|
| Canvas Explorer    | Grayed out, tooltip only   | Full UI visible, read-only       | Full UI, editable          |
| Voice Builder      | Hidden                     | Preview + Upgrade CTA            | Editable + savable modes   |
| Academy (Intermediate) | Inaccessible            | Visible index, gated entry       | Full access                |
| Implementor Tools  | Invisible                  | Preview tiles                    | Operational                |
| Phase 3 Menus      | Visible (always)           | Readable, gated if interactive   | Fully functional           |

---

### 💬 Gated Copy Samples
- “This feature is part of your next tier. You’ve already unlocked it—now claim it.”
- “You’ve shown the system how you think. Now let it respond at your level.”
- “Welcome to the edge of structure. Upgrade to shape it.”

---

### 🎯 Strategic Intent
- Transition from Beginner to Intermediate feels **earned, not sold**
- Upgrade prompts only appear **after curiosity is already engaged**
- ELos teaches first, reveals second, charges third

---

### 🧱 System Enforcement
- Tracked by Architect
- Indexed and stored by Librarian
- Personas only load intermediate scaffolds after unlock criteria met
- Builder UI elements are inert until upgrade
- **Phase 3 content always visible**, but feature triggers are upgrade-locked
