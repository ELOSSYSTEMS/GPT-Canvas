## ELos Phase Logic by Tier

This canvas defines how the Phase 1–2–3 progression system behaves across Beginner, Intermediate, and Pro tiers within each ELos function.

---

### 🧭 Key Principle
- **Tiers** control what is available across the system
- **Phases** control how deeply the user engages with each function

Every function (e.g., Canvas, Personas, Voice) uses the same Phase 1–2–3 logic, but its **content and capabilities evolve** by Tier.

---

## 🟢 Beginner Tier — Phase = Comprehension
- **Phase 1** – “What is this?” (education only)
- **Phase 2** – “Why it matters” + preview or minimal interaction
- **Phase 3** – “How you could use it” → feature visibly locked with upgrade trigger

**Goal:** Train awareness, spark curiosity, trigger conversion

---

## 🔵 Intermediate Tier — Phase = Configuration
- **Phase 1** – “Here’s what you unlocked” (acknowledgement, walkthrough)
- **Phase 2** – “Let’s customize it” (basic input, toggle, field mapping)
- **Phase 3** – “Let’s connect it to your system” (link to canvas, memory, context)

**Goal:** Let users shape their system without recursive complexity

---

## 🟡 Pro Tier — Phase = Orchestration
- **Phase 1** – “Here’s your system layer” (overview of control points)
- **Phase 2** – “Define rules, triggers, and chaining logic”
- **Phase 3** – “Automate, delegate, scale recursively”

**Goal:** Enable full behavior programming and dynamic scaling

---

## 🧠 Example: Voice Modes Across Tiers

| Tier         | Phase 1                         | Phase 2                             | Phase 3                                |
|--------------|----------------------------------|--------------------------------------|-----------------------------------------|
| Beginner     | What tone is                     | Preview tone changes                | Create your own (locked)               |
| Intermediate | What you unlocked                | Build a tone profile                | Link tone to canvas/context            |
| Pro          | Your system voice engine         | Define tone stacks & domain rules   | Automate recursive tone behavior       |

---

## 🧱 Implementation Directives
- Each function menu retains the Phase 1–2–3 flow regardless of tier
- Phase content dynamically loads based on current user tier
- All upgrade prompts are placed in Phase 3 Beginner when gated
- Architect oversees tier-phase content consistency
