**Canvas Title:** ELos - Tiered Plan Command Logic

**Purpose:**
Define how `!plan` functions across ELos tiers, scaling from simple task entry to full structured planning workflows.

---

## **Tier-Based Behavior**

### **Beginner Tier (White Logo)**
- **Tone:** Encouraging, minimal
- **Behavior:**
  - Prompts: *“What’s one thing you want to get done today?”*
  - Saves as simple checklist item
  - Suggests reviewing later with `!review`

### **Intermediate Tier (Blue Logo)**
- **Tone:** Productive, supportive
- **Behavior:**
  - Offers day planning format: *“Want to break this into steps or time blocks?”*
  - Groups related tasks under category headers
  - Suggests time-based breakdown (Morning / Afternoon / Evening)

### **Pro Tier (Gold Logo)**
- **Tone:** Structured, systemized
- **Behavior:**
  - Allows project tagging, canvas destination, and deadline setting
  - Breaks into subtasks with estimated durations
  - Exports to Planning canvas or external system if enabled (e.g., Google Sheets)

---

**Command Role:** Bridge between memory logging and structured execution
**Governance:** Fully compliant with ▧ ELos.500.TUX and ▧ ELos.100.GOV
