**Canvas Title:** ELos - Future Modules Overview

**Purpose:**
Consolidate and contextualize the advanced modules built to expand ELos' cognitive, observational, and system governance capabilities.

---

## **Module Index**

### 1. ▧ ELos.300.AUD – AI Auditing Toolkit
- Exposes assistant logic assumptions
- Logs behavioral state, persona stack, and memory cues
- Surface reasoning chain via `!audit`

**Recursive Leverage:**
- Feeds into Shadow Mode for passive audit
- Works in tandem with Diff Viewer for behavioral drift

### 2. ▧ ELos.300.DIF – Live Canvas Diff Viewer
- Compares any canvas across versions
- Highlights changes in logic, tone, or decisions
- Uses `!diff` to trigger view

**Recursive Leverage:**
- Integrates with AI Auditing to show assumption shifts over time
- Supports Memory Graph as diff overlay node

### 3. ▧ ELos.300.SHD – Shadow Mode
- Silently logs all system-level assistant behavior
- Exportable and timestamped
- Activated by `!shadowlog`

**Recursive Leverage:**
- Provides historical data for auditing, mapping, and training
- Shadow Mode + Auditor = passive QA pipeline

### 4. ▧ ELos.300.GPH – Memory Graph
- Visualizes canvases, commands, and persona relationships
- Interactively navigable knowledge map
- Activated via `!map`

**Recursive Leverage:**
- Surfaces behavioral hotspots or canvas clusters
- Integrated with Diff Viewer + Auditor for full traceability

---

## **Redundancy Check**

- **AI Auditing vs. Shadow Mode:** Shadow Mode logs **everything**, while Auditing surfaces **select** logic. They complement rather than duplicate.
- **Diff Viewer vs. Memory Graph:** Diff is version control; Graph is structure visualization. No overlap.
- **Auditor vs. Graph:** Graph shows what exists; Auditor tracks what’s *assumed*. Roles are distinct.

**Conclusion:** No functional redundancy. All modules are recursive-compatible and modular. Each fulfills a distinct diagnostic, navigational, or logging purpose.
