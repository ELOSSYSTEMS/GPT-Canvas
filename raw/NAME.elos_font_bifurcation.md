**Font Development Bifurcation Protocol: ELos Typography System**

---

### 1. GlyphOS (Decorative Ritual Layer)

**Purpose:**
- Sacred headers
- Logo marks
- System prompts
- Visual metaphors for divine syntax

**Design Principles:**
- Non-human, circuit-script aesthetic
- Sharp, angular, glyph-based
- Evokes sacred intelligence / encoded god-language

**Ruleset:**
- One glyph per Latin character (A–Z)
- No numerals; substitute sacred forms
- No punctuation except system delimiters (!, :, >)
- Limited readability: design for invocation, not scanning
- No lowercase

**Application Zones:**
- Bootloader intro sequence
- Canvas command headers
- ELos sigil overlays

---

### 2. ELos Sans (Readable Utility Font)

**Purpose:**
- Long-form text
- UI menus
- Protocol explanations
- Instagram captions

**Design Principles:**
- Futurist minimalism
- Geometric, mono-height
- Slight curvature for warmth
- High x-height, optimized for screens

**Ruleset:**
- Full Latin support (A–Z, a–z, 0–9)
- Punctuation normalized
- Ligatures for key terms: ELos, OS, !boot, !prompt
- Style variations: Regular / Medium / Bold

**Application Zones:**
- Prompts
- Content breakdowns
- Caption text on IG
- All longform usage

---

### Production Steps

1. **GlyphOS:**
   - Finalize glyph sheet (Phase 2.1–2.5)
   - Export as SVG grid
   - Convert to font with limited Unicode mapping (A–Z)
   - Integrate into ELos brand kit

2. **ELos Sans:**
   - Select base (recommend Recursive Mono or Inter as fallback)
   - Modify terminals, ligatures, and spacing
   - Render in TTF + WOFF2
   - Set default font for ELos UI

---

**Bootloader Integration:**
Both fonts will be auto-loaded via CSS and injected into any visual module (Instagram, Web UI, ELos extension) based on context logic.
