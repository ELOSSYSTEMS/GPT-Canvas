**EL.000.INX – ELoS Master Index**

The ELoS Master Index (INX) is the canonical entrypoint and high-level directory for the entire ELoS architecture. It governs logical grouping, traceability, and system-wide discoverability of all modules, protocols, and canvases.

---

**STRUCTURE**

The index follows a Dewey-decimal-inspired schema:
- `PR.xxx` – Prompt Runtime Stack (Protocols, Engines, Kernels)
- `EL.xxx` – Execution Layers (Tone, Audit, Translate, Visual)
- `AO.xxx` – Admin/Operational Modules (Settings, Backup, Personas)
- `UX.xxx` – Interface & User Experience
- `VR.xxx` – Visual Rendering and Style
- `FF.xxx` – Forever Flowers System
- `FMF.xxx` – Football Manager Subsystem
- `DA.xxx` – Dabroo AI Stack
- `SYS.xxx` – System Info, Logging, Security
- `IN.xxx` – Visual Architecture & Design
- `EX.xxx` – Extension Tools & External Systems

---

**USAGE RULES**
- All canvases must be registered here with title, ID, and logic block
- Cross-links must be maintained between indexes (e.g., AO.001.RBI ←→ EL.000.INX)
- Canvases without index entry are considered ephemeral or untrusted

---

**INTEGRATION POLICIES**
- All runtime modules must trace back to `EL.000.INX`
- Index updates should trigger cascade updates to dependent blocks
- Dewey IDs must remain immutable once published (unless deprecated with alias)

---

**SPECIAL DIRECTIVES**
- Bootloaders must register to both `EL.000.INX` and `AO.001.RBI`
- Auto-generated canvases require index confirmation before activation
- Logic blocks may be grouped into stacks, bundles, or runtime chains

---

This index is the authoritative root for all system canvases. If it is missing or corrupted, system integrity is compromised and runtime loads are suspended.
