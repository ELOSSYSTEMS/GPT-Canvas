## 1. Non-Negotiable Design Doctrine

The site is **not** an experience to explore.

It is a **competent professional stepping in quietly**.

If a customer notices the UX, the system failed.

Desired internal reaction:
> “ברור. זה הגיוני.”

Not:
- “וואו איזה אתר”  
- “איזו חוויה”  
- “מעניין”

---

## 2. Israeli Luxury Market Constraints (Accepted Facts)

These constraints override global best practices.

- Polish creates suspicion.
- Explanation creates resistance.
- Visible hierarchy creates rejection.
- Choice creates negotiation.
- Defaults create authority.
- Silence signals confidence.

Therefore:
- Fewer screens beat better screens.
- Less copy beats better copy.
- Fewer options beat smarter options.

---

## 3. Structural Principles (LOCKED)

### 3.1 Situational Entry Beats Categories

The homepage **never** opens with:
- Collections
- Grids
- Filters
- Inspiration

Opening job:
**Identify the user’s situation and route immediately.**

---

### 3.2 One Job per Screen

Every screen answers one question only:
- “What is this?”
- “Is this for me?”
- “What happens next?”

No screen may perform multiple roles.

---

### 3.3 Default Equals Authority

Anything optional feels negotiable.

Therefore:
- Defaults are preselected.
- Choices are secondary.
- Overrides are hidden, not celebrated.

---

### 3.4 Tier Separation Is Mandatory

Users must **never** feel they are being sold upward.

Rules:
- Tier 1 users do not see Tier 3.
- Tier 3 is invisible unless explicitly entered.
- No comparisons. No ladders. No “upgrade” language.

---

## 4. Canonical Information Architecture

### 4.1 Global Navigation (Minimal)

- Logo (tap = home)
- Cart icon

No menu bar.
No category navigation.

Optional bottom navigation (mobile):
- בית
- סידורים
- סל

Only if needed for orientation.

---

## 5. Homepage Wireframe (Triage Screen)

### Screen Purpose
Route the user without making them think.

### Structure (Top to Bottom)

1. **Single Hero Image**  
   Real home. Real scale. No styling theatrics.

2. **One Sentence Only**  
   Neutral, competent tone.

3. **Situational Selector (3 Cards)**

   - רוצה לראות איך זה מרגיש בבית  
     → Entry Tier (Fast Path)

   - הבית צריך נוכחות שקטה  
     → Tier 2 (Studio-Picked)

   - יש חלל שדורש פתרון מדויק  
     → Tier 3 (Custom)

No prices.
No explanations.

4. **Exit Opportunity**  
   User can scroll or leave without penalty.

Nothing else lives on the homepage.

---

## 6. Tier 1 — Entry Path (Fast Conversion)

### Purpose
Enable trial with **near-zero cognitive load**.

### Behavioral Goal
Conversion in **<30 seconds**.

### Wireframe Rules

- No storytelling above the fold.
- No education before commitment.
- No future possibilities visible.

### Flow

1. Product shown with default configuration
2. Optional color change (secondary)
3. Add to Cart
4. Gift question (post-commitment)
5. Checkout

### What Must Not Exist

- Tier language
- Design explanations
- Custom processes
- Upsells before cart

---

## 7. Tier 2 — Studio-Picked Path

### Purpose
Provide quiet, curated design confidence.

### Positioning
Not “more expensive”.
Not “premium”.

Instead:
**Chosen by the studio.**

### Wireframe Rules

- Maximum 3–5 items visible at once.
- No sorting.
- No filters.
- No grids.

### PDP Structure

1. Image
2. One-line intent
3. Price
4. Add to Cart
5. Studio Finish note (collapsed)

Everything else is secondary.

---

## 8. Tier 3 — Custom Entry (Emotion First)

### Purpose
Invite collaboration without intimidation.

### Entry Screen

Question only:
> “מה חסר לך בחלל הזה?”

Options:
- שקט
- חום
- נוכחות
- דיוק

No prices.
No process explanation.

### Only After Selection

- Explain collaboration
- Show that guidance is shared
- Emphasize responsibility transfer to studio

### Exit Rule
User can abandon at any step without consequence.

---

## 9. Global UX Rules (Israel-Specific)

- No animations that draw attention to themselves
- No progress bars
- No wizards
- No feature callouts
- No value stacking

Layouts repeat intentionally.
Sameness = trust.

---

## 10. What Must Never Be Built

This list is **explicitly forbidden**:

- Homepage product grids
- “Choose your journey” language
- Tier comparison tables
- Badges like Premium / Exclusive / Signature
- Educational pages about dried flowers
- Feature lists
- Early upsells
- Visible hierarchy

If any of the above appears, the rebuild has failed.

---

## 11. System Mapping Requirement

Every screen must map cleanly to:
- A Next.js route
- A single React component
- A single operational outcome

No conceptual screens.
No marketing-only pages.
