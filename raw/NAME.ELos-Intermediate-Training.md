**ELos Intermediate System Training: From Operator to Architect**

**Tier:** Intermediate  
**Audience:** Users who understand basic prompt structure and threading, and now want to *design reusable systems and assistant workflows*.  
**Tone:** Blunt, systems-driven, no appeasement — assumes user wants precision and leverage.

---

## Objective
Move from tactical prompting to structured assistant-building. Begin modular thinking, command chaining, and memory simulation.

---

## Module Breakdown

### 01. Modular Prompt Systems
**Goal:** Build prompt templates that scale.

- Role-based prompt libraries
- Token budgeting: brevity vs depth
- Output predictability and constraints

**Practice:** Create a set of 3 prompts for the same task: short-form, long-form, and reusable system message.

---

### 02. Persona Simulation and Role Injection
**Goal:** Program behavior, tone, and expertise.

- Define tone, function, constraint in persona
- Chain behaviors (e.g., analyst + critic + summarizer)
- Reusability across tasks

**Practice:** Design a custom assistant for a recurring task (e.g., product copywriter).

---

### 03. !Commands System Logic
**Goal:** Introduce command-based interface structure.

- Supported commands (!overview, !rewrite, !explain, !store)
- Syntax logic and chaining
- When to use vs when to prompt naturally

**Practice:** Execute 3 chained !commands in a single interaction.

---

### 04. Canvas Memory Simulation
**Goal:** Build continuity without real-time memory.

- Simulated memory via canvases (!store → !recall)
- Indexing and linking thoughts
- Assistant memory roles (scribe, librarian, auditor)

**Practice:** Create a persona + canvas + recall chain for a topic.

---

### 05. Structural UX Patterns
**Goal:** Use ChatGPT like a custom OS.

- Working in tabs (conversations = processes)
- Naming, versioning, and modular storage
- Switching assistant roles based on task layer

**Practice:** Build a workflow using 3 tabs + 2 personas + 1 canvas.

---

## Transition to Pro Tier
Once users:
- Use !commands fluently
- Build custom assistants with structured behavior
- Simulate memory through canvases

They are eligible for Pro: Full assistant governance, tone shifts, automation logic, and advanced UX overlays.
