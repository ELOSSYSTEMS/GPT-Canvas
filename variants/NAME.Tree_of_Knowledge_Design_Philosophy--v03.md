## 🌳 Tree of Knowledge Onboarding Masterplan (ELos)

### Core Philosophy
The Tree of Knowledge is a **living, evolving interface** designed to educate and empower users through **progressive self-awareness and system integration**. It avoids gamification and instead uses **natural metaphors** (growth, light, color) to reflect learning, mastery, and system depth. It is not a dashboard. It is an ecosystem.

---

### 🔄 Three-Phase Growth Model (Per Menu / Leaf)
Each branch/menu in the Tree progresses through **three discrete phases**, **independent of user progress in other branches**:

- **Phase 1: White Leaf (Awareness)**
  - Introduces the *ChatGPT-native function* simply: "This is what this part of ChatGPT does."
  - Must be interactive to maintain engagement—e.g., prompt the user to run a diagnostic, display a self-check summary, or view a short system animation.
  - Unlocks Phase 2 only after a minimal interaction (e.g., "Show me my current voice settings").

- **Phase 2: Blue Leaf (Understanding)**
  - Activated once Phase 1 is interacted with.
  - Adds deeper context, editable fields, and system-level flexibility.
  - **This is where ELos is introduced.**
  - Includes a **Book Icon (📚)** to open a dedicated **ELos Academy resource**, tiered by level: Beginner / Intermediate / Pro.
  - Focus: "Here’s how you can modify and understand this with ELos."

- **Phase 3: Gold Leaf (Practical Application)**
  - Activated after meaningful action in Phase 2.
  - Emphasis shifts from exploration to application.
  - **No loop-building in beginner tier.**
  - Options include:
    - [🖊️ Create a Canvas: "Voice Rules"]
    - [📓 Save this tone as a reusable voice profile]
    - [🌱 Clone and Adapt Voice for New Domain]
  - Focus: "You can now *store and structure* this knowledge."

---

### 🌿 Example Menu: “ChatGPT Voice Mode”

#### Phase 1 (White)
> “This is how I currently speak to you: Blunt. Socratic. No fluff.  
Want to see a preview or try another tone?”

- [🎤 Hear Voice Sample]  
- [🔍 Show Current Settings]  
- [👍 Got it]

---

#### Phase 2 (Blue)
> “Tone isn’t just about words. It’s about how I challenge your thinking. Let’s tune it.”

Options:
- ✅ Adjust assertiveness
- ✅ Toggle Socratic vs. directive modes
- ✅ Add context-based voice shifts (e.g., Finance vs. Personal)
- 📚 [Open Beginner Guide to Voice Modes] *(ELos Academy canvas)*

---

#### Phase 3 (Gold)
> “You’ve refined how I speak. Want to formalize it as a reusable rule?”

- [🖊️ Create a Canvas: "Voice Rules"]
- [📓 Save this tone as a reusable voice profile]
- [🌱 Clone and Adapt Voice for New Domain]

---

### 🌲 Color Metaphor (System-Wide)
- **White** = New / dormant / untouched
- **Blue** = Explored / tuned / growing
- **Gold** = Mastered / structured / actionable

Leaves change colors per menu-phase, not by total progress. The **tree evolves organically**, not by point collection.

---

### 🔹 Visual and UX Principles
- The menu is alive: it adapts, grows, and shows history.
- Users should feel like they are *cultivating* an interface, not clicking through software.
- The OS becomes more visually rich and structurally flexible as understanding deepens.
- New branches do not appear based on "gamified" conditions but emerge naturally from interaction and reflection.

---

### 🔹 ELos Academy Integration
- Each menu item in **Phase 2** includes a **Book Icon (📚)** linking to an **educational canvas**.
- Canvases are categorized by user level: Beginner / Intermediate / Pro.
- Stored as persistent, modular knowledge—available via canvas library or embedded in menu flows.
- Format can be `.txt`, `.md`, `.pdf`, or local canvas object.
- Designed to be used **as a learning scaffold**—users can explore before editing or building.
